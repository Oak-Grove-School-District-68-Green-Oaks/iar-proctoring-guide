⚠ Known Issues & Reminders

▼
Restart Devices the Morning of the Test
Posted March 31, 2026

Both testing devices and proctor devices must be restarted the morning of each test session. Not sleep, not close-the-lid — a full restart. This clears background processes that can interfere with TestNav's kiosk mode and prevents the majority of mid-test crashes.

MacBook Users: Launch TestNav from the Applications Folder
Posted March 31, 2026

Students on MacBooks must open the TestNav app from the Applications folder (Finder → Applications → TestNav). Do not let students search for "TestNav" in a browser. If the app is missing from a device, Google will take them to the TestNav website, which looks legitimate and will let them sign in — but it is not the correct testing environment and their session will not count. If any device is missing the TestNav application, contact the tech team immediately rather than using the website.

1 What You'll Need

  • A computer to proctor with (separate from student devices)
  • The written scripts to be read aloud to students
  • Student testing tickets, each containing:
    • — Student name
    • — Last name (entered at testing)
    • — State student number (the identifier)
  • Your proctor card, which includes:
    • — Test Code
    • — Proctor Password
    • — Seal Codes (one per section)

2 Before Test Day

Strongly recommended: Have students launch TestNav before test day to complete these checks. Resolving issues during the test delays the entire class.
1

Restart the device. A full restart, not just sleep or closing the lid.

2

Open the TestNav application from the Applications folder. Do not search for TestNav in a browser.

3

Select the correct customer. Click the icon in the top-right corner, choose "Choose a different customer," then select Illinois.

4

Test audio and microphone. Students should allow microphone access when prompted.

Important: If a student denies microphone access, TestNav will likely not allow the test to proceed. The only fix is to go into the device's Security & Privacy settings, which significantly delays testing.

3 Proctor Setup

1

Go to il.adamexam.com/#/proctor on your proctor computer.

2

Enter the Test Code and Proctor Password from your proctor card. Click Submit, then Confirm.

3

Enter your first and last name, check the "I Agree" box on the Proctor Acknowledgment, and click Save.

4

You're now in the proctoring platform. Display the Test Code for students — write it on the board or use the Display button in the top-right corner to project it.

4 Student Login

1

Students restart their devices, then open the TestNav app from the Applications folder.

2

Students enter the Test Code displayed on the board.

3

Students enter their last name and state identifier from their testing ticket, then click Confirm.

4

On your proctor dashboard, press Refresh (or enable Auto Refresh) until all students appear with a "Needs Attention" status. The page does not update in real time.

Note: Auto Refresh only updates every 5 minutes. Use the manual Refresh button if you need a faster update.

5 Running the Test

1

Once all students appear in the list, go to Group Actions and click "Approve All Sessions", then confirm.

2

Student devices will prompt for App Self-Lock (kiosk mode). Students must tap "Yes" to proceed.

3

Provide the Seal Code for the current section. Display it on the board or read it aloud.

Do not give out Seal Codes early. Only provide a Seal Code when your entire class is ready to move to that section.
4

Monitor progress using the Refresh button. You can track Section Progress, Item Progress, and Test Duration for each student.

6 Wrapping Up

1

When all students have submitted their section, go to Group Actions and click "Exit All Sessions."

2

Students can now quit TestNav.

3

Repeat this entire process for each remaining section of the IAR until the assessment is complete.

If Something Goes Wrong — Reseating a Student

If a student is exited from the test for any reason — accidental exit, a crash, or a "Session in Use" message — you can get them back in. This is the most common issue during testing.

1

Tell the student to stop and wait. Don't let them try to log back in yet.

2

Refresh your proctor dashboard so all student statuses are current.

3

Click the three-dot menu (⋮) next to the student's name and select "Reseat Session." Confirm when prompted.

4

Wait for the dashboard to show "Reseat" status for that student. Refresh again if needed.

What to look for: The student's Test Status column will change to show the Reseat badge. You may need to Refresh again to see the update.
5

Now tell the student to log back into TestNav. Their test will resume where they left off.

What the student sees: If a "Session in Use" message appears, the student should click Retry. Their test will resume exactly where they left off — no work is lost.
Tip: Keep the Refresh button handy throughout testing. The dashboard does not update in real time, and catching an "Exited" status early means less lost testing time.
